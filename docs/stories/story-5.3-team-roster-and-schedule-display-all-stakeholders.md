# Story 5.3: Team Roster & Schedule Display (All Stakeholders)

**Status**: Draft

**Story**
**As a** Player or Parent,
**I want** to easily view the team roster and schedule,
**so that** I am always informed about who is on the team and when our events are.

**Acceptance Criteria**
1.  Players and Parents can view the current team roster, including player names and assigned team.
2.  The team roster view allows for filtering/sorting (e.g., by jersey number, position, if available).
3.  Players and Parents can view the full team schedule (games, practices, events) in a clear, accessible format.
4.  Schedule entries display date, time, location, and event type.
5.  Communication from coaches (e.g., coaching vision, team guidelines) is accessible for parents.

**Tasks / Subtasks**
- [ ] **Task 1: Roster and Schedule UI (AC: #1, #2, #3, #4, #5)**
    - [ ] Design the interface for viewing the team roster and schedule.
    - [ ] Implement the React component for the roster and schedule view.
- [ ] **Task 2: Backend for Roster and Schedule (AC: #1, #2, #3, #4, #5)**
    - [ ] Create API endpoints to fetch the data for the roster and schedule view.
    - [ ] Implement the business logic for retrieving roster and schedule data.

**Dev Notes**
*   **Read-Only**: This view is read-only for players and parents.
*   **Clear and Simple**: The interface should be very clear and simple to use.
*   **Combined View**: Consider a combined view that shows the roster and schedule together.

**Testing**
*   Unit tests for the new API endpoints.
*   Component tests for the roster and schedule view.
